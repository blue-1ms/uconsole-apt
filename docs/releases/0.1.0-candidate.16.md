# uConsole kernel 7.1.3-1003 stable

This is the first kernel-only release to pass the complete CM4 Lite, microSD, CWU50 display, and
A/B rollback gates. The exact historical tag remains `0.1.0-candidate.16` so that promotion does
not alter the validated bytes.

## Contents

- Runtime kernel: `7.1.3-1003-uconsole`
- Package version: `7.1.3-1003.1.uconsole.1`
- Platform package used for the original validation: `uconsole-platform 0.1.15`
- Patched source commit: `4bad9ea6781a9dde63322956bba4c10070957aaf`
- Offline bundle: `uconsole-apt-repository-0.1.0-candidate.16.tar.zst`
- Bundle SHA-256:
  `59530b18a9e9dcc692bf3b7c5af62ac678ec6e6c7d16d5543478a696655c93ca`

This release includes the audited CM4 Lite lean configuration, CWU50/OCP8178/AXP support, the
complete Raspberry Pi overlay seed, speaker/headphone routing, and the BCM2711 V3D low-address
DMA fix. Hardware validation observed zero SWIOTLB warnings, DMA mapping errors, and GPU/DRM
resets.

## Validation

The same package bytes passed:

- package identity, ownership, ABI, modules, headers, buildinfo, DTB/DTBO, initramfs,
  external-module, and signed APT repository validation;
- one cold boot, two warm reboots, and `piboot-try` promotion;
- a real `new=bad` automatic rollback to `7.1.3-1001-uconsole`, followed by retry and promotion
  of the same bytes;
- panel, backlight, DRM, keyboard/game keys, trackball, power key, speaker/headphone,
  PMIC/battery, Wi-Fi, and Bluetooth;
- USB storage, RTL8153 USB Ethernet, and a USB microphone;
- the Mac-readable FAT diagnostic mailbox and privacy scan.

Detailed evidence is stored in the
[`uconsole-ubuntu-lts` matching tag](https://github.com/blue-1ms/uconsole-ubuntu-lts/tree/0.1.0-candidate.16/docs/releases/0.1.0-candidate.16).

## Installation

For a machine that has never used this repository, complete the
[first-install bootstrap](https://github.com/blue-1ms/uconsole-apt#first-installation) before
installing packages.

The preferred stable installation is now:

```bash
sudo apt-get update
sudo apt-get install --yes uconsole-platform
sudo uconsole-kernel-policy-validate
sudo apt-get install --yes uconsole-kernel uconsole-plymouth-theme
sudo reboot
```

Allow the automatic restarts required by `piboot-try`. Never modify `/boot`, `/boot/firmware`,
kernel images, DTB/DTBO files, or slot state manually.

## Rollback

The validated N-1 kernel is `7.1.3-1001-uconsole`. If validation fails, the final slot diagnostic
is written to the FAT partition before the device returns to N-1. Do not remove N-1 until the new
kernel has completed promotion, cold/warm boot validation, and a controlled rollback test.
